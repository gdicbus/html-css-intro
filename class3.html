Content Sectioning Elements

Right now our websites are just a bunch of content on a page.

As we get closer to our goal of a fully-styled website, we need to start thinking about how our content is organized.

Content Sectioning Elements

Header

Tag(s):
<header></header>
Category:
Content sectioning
Acceptable content:
Header-related elements like <h1> - <h6>, logos, and author attributions.

A page can have as many <header> elements as necessary.

While they're helpful for wrapping and styling headings (<h1> - <h6>), they are not required.

<header>
  <h1>A Happy Lorem Ipsum</h1>
  <h2>Bob Ross quotes generator</h2>
</header>

Main

Tag(s):
<main></main>
Category:
Content sectioning
Acceptable content:
Any content that is unique to that page.

A document must not have more than one <main> element. Structures that repeat across the website (such as navigation or copyright footers) should not be inside the <main> element.

Main

<main>
  <h1>In Life, You Need Colors</h1>
  <p>
    I'm gonna start with a little Alizarin crimson
    and a touch of Prussian blue. We’ll use
    Permanent Red. Brown is such a nice color.
  </p>
  <h2>Painting Should Do One Thing:</h2>
  <ol>
    <li>It should put happiness in your heart.</li>
  </ol>
</main>

Nav

Tag(s):
<nav></nav>
Category:
Content sectioning
Acceptable content:
Navigational links

This element is intended to mark the presence of navigational content, primarily within a site.

There can be multiple <nav> elements on a single page.

Nav

Example: Global Navigation

<nav>
  <ul>
    <li><a href="index.html">Home</a></li>
    <li><a href="about.html">About</a></li>
    <li><a href="contact.html">Contact</a></li>
  </ul>
</nav>

With styles, could look like:

  • Home
  • About
  • Contact

Nav

Example: Breadcrumb Navigation

<nav>
  <ol>
    <li>
      <a href="brand.html">Brands</a>
    </li>
    <li>
      <a href="brand/bobross.html">Bob Ross</a>
    </li>
    <li>
      <a href="brand/bobross/brushes.html">Brushes</a>
    </li>
  </ol>
</nav>

With styles, could look like:

  • Brands » 
  • Bob Ross » 
  • Brushes

Footer

Tag(s):
<footer></footer>
Category:
Content sectioning
Acceptable content:
Any except <header> and <footer>

Typically contains copyright and author information. Can also contain duplicate global navigation (helpful on long pages).

Footer

<footer>
  <nav>
    <ul>
      <li><a href="index.html">Home</a></li>
      <li><a href="about.html">About</a></li>
      <li><a href="contact.html">Contact</a></li>
    </ul>
  </nav>
  <p>Copyright © Grace Hopper</p>
</footer>

The Box Model

Every element in HTML is a rectangle that has 3 built-in, default CSS properties: padding, border, & margin.

Border

Applying a border to an element in CSS is akin to drawing a line around it.

There are several border-related properties:

border-color, border-style, border-width, border-radius, and border-image

We won't be covering border-image in this class, but you can learn about more about it here.

Border

Style and Width

Elements must have the border-style and border-width properties defined in order for the border to be visible. In this example, each element's border-width is set to 10px to demonstrate each border-style:

Border

Color

The border-color property sets the color of the border. It takes all of the same color values we covered in Class 2.

Border

Radius

CSS can handle rounded corners! Note how the corners are still round even when the border-style is set to hidden or none.

Note: both the radius and width are set to 10px here. Setting the radius to a higher value than the width can cause certain border styles look wonky.

Border

Shorthand

The following styles...

p {
  border-color: red;
  border-style: dotted;
  border-width: 9px;
  border-radius: 5px;
}

... can also be written (mostly) in shorthand.

p {
  border: 9px solid red;
  border-radius: 5px;
}

Note: order matters for border shorthand. Width, style, color; in that order. Radius is not included.

Padding

Space between the border and the content

Padding

Notice that the padding adds to the total width of the box.

Padding

15 pixels on all sides

padding: 15px;

10 pixels on top only

padding-top: 10px;

10 on top, 5 on right, 3 on bottom, 5 on left

padding: 10px 5px 3px 5px;

Padding

Four values 

padding: top right bottom left;

Two values

padding: top/bottom right/left;

One value

padding: all;

Padding

padding: 10px 20px 30px 40px;

Border

The edge around the box.

Border

Borders are specified as "thickness, style, color."

A solid red border

border: 1px solid #ff0000;

A thick dotted black top border

border-top: 4px dotted #000000;

Two different border styles

border-top: 1px solid #ff0000;
border-bottom: 4px dotted #000000;

Border - Other Properties

border-width: 10px;
border-style: dashed;
border-color: #666666;

You can specify each property separately, or all three together.

Margin

The transparent area around the box that separates it from other elements.

Margin

15 pixels on all sides

margin: 15px;

10 on top, 5 on right, 3 on bottom, 5 on left

margin: 10px 5px 3px 5px;

10 pixels on top

margin-top: 10px;

Auto Margin

If a margin is set to auto on a box that has width, it will take up as much space as possible.

CENTERED

margin: auto;
width: 300px;

FLUSH-RIGHT

margin-left: auto;
margin-right: 5px;
width: 300px;

Pseudo-classess

Changing the format of a link when you hover over it is accomplished by using pseudo-classes.

CSS pseudo-classes are used to add special effects to some selectors.

Syntax:

selector:pseudo-class{
    property:value;
  }

Example:

a:link{
    text-decoration: none;
  }

Pseudo-classes

a:link  {color:#FF0000;} /* unvisited link */
  a:visited {color:#00FF00;} /* visited link */
  a:hover, a:focus {color:#FF00FF;} /* mouse over or select with keyboard*/
  a:active {color:#0000FF;} /* selected link */

a:hover MUST come after a:link and a:visited in the CSS definition to be effective!

a:active MUST come after a:hover in the CSS definition to be effective!

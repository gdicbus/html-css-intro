Content Sectioning Elements

Right now our websites are just a bunch of content on a page.

As we get closer to our goal of a fully-styled website, we need to start thinking about how our content is organized.

Content Sectioning Elements

Header

Tag(s):
<header></header>
Category:
Content sectioning
Acceptable content:
Header-related elements like <h1> - <h6>, logos, and author attributions.

A page can have as many <header> elements as necessary.

While they're helpful for wrapping and styling headings (<h1> - <h6>), they are not required.

<header>
  <h1>A Happy Lorem Ipsum</h1>
  <h2>Bob Ross quotes generator</h2>
</header>

Main

Tag(s):
<main></main>
Category:
Content sectioning
Acceptable content:
Any content that is unique to that page.

A document must not have more than one <main> element. Structures that repeat across the website (such as navigation or copyright footers) should not be inside the <main> element.

Main

<main>
  <h1>In Life, You Need Colors</h1>
  <p>
    I'm gonna start with a little Alizarin crimson
    and a touch of Prussian blue. We’ll use
    Permanent Red. Brown is such a nice color.
  </p>
  <h2>Painting Should Do One Thing:</h2>
  <ol>
    <li>It should put happiness in your heart.</li>
  </ol>
</main>

Nav

Tag(s):
<nav></nav>
Category:
Content sectioning
Acceptable content:
Navigational links

This element is intended to mark the presence of navigational content, primarily within a site.

There can be multiple <nav> elements on a single page.

Nav

Example: Global Navigation

<nav>
  <ul>
    <li><a href="index.html">Home</a></li>
    <li><a href="about.html">About</a></li>
    <li><a href="contact.html">Contact</a></li>
  </ul>
</nav>

With styles, could look like:

  • Home
  • About
  • Contact

Nav

Example: Breadcrumb Navigation

<nav>
  <ol>
    <li>
      <a href="brand.html">Brands</a>
    </li>
    <li>
      <a href="brand/bobross.html">Bob Ross</a>
    </li>
    <li>

  • Brands »

  • Brushes

Footer

Tag(s):
<footer></footer>
Category:
Content sectioning
Acceptable content:
Any except <header> and <footer>

Typically contains copyright and author information. Can also contain duplicate global navigation (helpful on long pages).

Borders

Applying a border to an element in CSS is akin to drawing a line around it.

There are several border-related properties:

border-color, border-style, border-width, border-radius, and border-image

We won't be covering border-image in this class, but you can learn about more about it here.

Border Properties: Width

The width of the border (or line thickness) is editable.

p {
  border-width: 10px;
}

Border Properties: Style

Borders don't have to be solid lines. CSS has several different types of border-styles to choose from. 

p {
  border-style: dotted;
}

Border Properties: Color

The border-color property sets the color of the border. It takes all of the same color values we covered in Class 2.

p {
  border-color: red;
}

Border Shorthand

The following styles...

p {
  border-color: red;
  border-style: dotted;
  border-width: 9px;
}

... can also be written in shorthand.

p {
  border: 9px dotted red;
}

The properties can be listed in any order.

Border: Styling Individual Sides

We can also apply differently-styled borders to each side of an element.

p {
  border-left: 10px solid lightblue;
  border-top: 30px double thistle;
  border-right: none;
  border-bottom: 15px ridge teal;
}

The above styles produce this result:

Border Properties: Radius

CSS can handle rounded corners! It works even when the border-style is set to hidden or none.

p {
  border-radius: 10px;
}

Note: Setting the radius to a higher value than the width can cause the more complex border-styles to look broken.

Border Radius: Individual Corners

Rounded corners can be applied individually using the following properties:

p {
  border-top-left-radius: 5px;
  border-top-right-radius: 10px;
  border-bottom-right-radius: 25px;
  border-bottom-left-radius: 40px;
}

Border Radius: Shorthand

We can also apply the radius styles in shorthand via the border-radius property.

p {
  border-top-left-radius: 5px;
  border-top-right-radius: 10px;
  border-bottom-right-radius: 25px;
  border-bottom-left-radius: 40px;
}
p {
  border-radius: 5px 10px 25px 40px;
}

In shorthand, the order of the values matters. When there are 4 values provided, they are assigned clockwise, starting from the top left.

Border Radius: Shorthand

When two values are provided:

border-radius: 5px 10px;

The first (5px) is applied to top-left and bottom-right.
The second (10px) is applied to top-right and bottom-left.

When three values are provided:

border-radius: 5px 10px 2px;

The first (5px) is applied to top-left.
The second (10px) is applied to top-right and bottom-left.
The third (2px) is applied to bottom-right.

Let's Develop It!

Play around with the border and border-radius properties!

  • Try different colors, weights and styles.
  • Apply a border to only one or two sides of an element.
  • Use shorthand to assign differently-rounded corners to a single element.

The Box Model

Aside from border, every element in HTML has two other built in CSS properties: padding and margin.

The combination of these properties and how they interact is referred to as The Box Model.

Padding

Padding is the area between an element's content and its border.


Most elements have a default of no padding applied:

Happy little example paragraph.



The following CSS adds 20px of padding to each side:

p {
  padding: 20px;
}

Happy little example paragraph.

Padding: Individual Sides

As with borders, we can add different amounts of padding to each side of an element.

p {
  padding-top: 10px;
  padding-right: 20px;
  padding-bottom: 30px;
  padding-left: 40px;
}

Padding: Shorthand

We can also apply these styles in shorthand via the padding property.

p {
  padding-top: 10px;
  padding-right: 20px;
  padding-bottom: 30px;
  padding-left: 40px;
}
p {
  padding: 10px 20px 30px 40px;
}

The order of the values matters. When there are 4 values provided, they are assigned clockwise, starting from the top.

Padding: Shorthand

When two values are provided:

padding: 5px 10px;

The first (5px) is applied to top and bottom.
The second (10px) is applied to right and left.

When three values are provided:

padding: 5px 10px 2px;

The first (5px) is applied to top.
The second (10px) is applied to right and left.
The third (2px) is applied to bottom.

Margin

Margin is the transparent area around an element that separates it from other elements.

Margin

Unlike padding, many elements do have some margin applied by default. Headings (h1-h6) and paragraphs are excellent examples.

We can use the Developer Tools to "highlight" the margins and see how they interact.

Margin: Shorthand

As with padding, we can apply margin in shorthand.

p {
  margin-top: 10px;
  margin-right: 20px;
  margin-bottom: 30px;
  margin-left: 40px;
}
p {
  margin: 10px 20px 30px 40px;
}

The order of the values matters. When there are 4 values provided, they are assigned clockwise, starting from the top.

Margin: Shorthand

When two values are provided:

margin: 5px 10px;

The first (5px) is applied to top and bottom.
The second (10px) is applied to right and left.

When three values are provided:

margin: 5px 10px 2px;

The first (5px) is applied to top.
The second (10px) is applied to right and left.
The third (2px) is applied to bottom.

Margin: Collapse

When two vertically adjacent elements have margin applied, their top and bottom margins will "collapse".

Given the following code...

h1 {
  margin-bottom: 30px;
}

p {
  margin-top: 10px;
}
<h1>
  Hello world.
</h1>
<p>
  Welcome to my website.
</p>

...while it might seem like there should be a total of 40px of margin between these elements, there will only be 30px.

The Box Model & Inline Elements

Inline elements (eg. span, em, strong, etc.) don't play as nicely with the box model as block elements do.

While block elements grow downward and right-ward from their top-left corner, inline elements stay anchored to the baseline of their content text. This can cause all sorts of issues.

Pseudo-classes

A CSS pseudo-class is a keyword added to a selector that allows us to style an element based on its state.

In this course, we'll focus solely on the pseudo-classes associated with the anchor element. You can learn about the others here.

Pseudo-classes

We know how to style the anchor element, but what if we want it to have different styles when a user hovers over it with their cursor? That's where pseudo-classes come in.

a {
  color: teal;
}

a:hover {
  color: limegreen;
}

Pseudo-classes

In CSS, we use the colon (:) to signify that we're declaring a pseudo-class. There should be no space between the selector that we're styling and the pseudo-class. The lack of a space (also known as "concatination") tells the browser that we're looking for the state on that specific element.

/* Correct */
a:hover {
  color: limegreen;
}

/* Incorrect */
a: hover {
  color: limegreen;
}

Pseudo-classes

a:link {
  color: blue;
}

Any anchor that has not yet been visited (clicked).

a:visited {
  color: purple;
}

Any anchor that has been visited (clicked).

a:hover, a:focus {
  color: limegreen;
}

Any anchor that is being hovered over by a mouse or is in focus (selected via the tab key).

a:active {
  color: pink;
}

Any anchor that is being selected (pressed).

Pseudo-classes

Some anchor cascade "gotchas":


The a:hover rule must come after the a:link and a:visited rules in the CSS to be effective.


The a:active rule must come after the a:hover rule in the CSS to be effective.

Text-Align

In block elements (h1, p, etc.) we can set the alignment of the text within the element.

h1 {
  text-align: right;
}

p {
  text-align: center;
}

p.justify {
  text-align: justify;
}

Comments

We can comment out lines of code in CSS, too!

/* Hello, I am a comment. */
a:hover {
  color: limegreen;
}

/* a:hover {
  color: limegreen;
} */

a:hover {
  color: thistle;
  /* text-decoration: underline; */
}

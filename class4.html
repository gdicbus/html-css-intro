Width

An element's width describes the width of the content. Padding and border are additional measurements.

header {
  width: 100%;
  padding: 1%;
}

The overall width of this header would be 102%.

Box-Sizing

content-box

This additive effect is caused by a CSS property called box-sizing, whose default value is content-box. This means that the size of the box (in this case, the width) is going to be applied to the content.

Box-Sizing

content-box

There are two ways to deal with calculating an element's width and padding. You can either do the mental math and apply the width and padding accordingly...

header {
  width: 80%;
  padding: 10%;
}

80% + 10% + 10% = 100%

Box-Sizing

border-box

Or you can reset the box-sizing property altogether! The border-box value tells the browser to apply the width to the combination of the border, padding, and the width.

header {
  box-sizing: border-box;
  width: 80%;
  padding: 10%;
}

80% + 10% + 10% = 100%

Property: Height

Sets the height of an element.

Does not include padding or borders. Remember to add these to the height.

p.alert{
      height: 50px;
      padding: 5px;
      border: 2px solid red; //Total height is 64px
    }

Generic Elements

We've learned about lots of elements that were created for very specific purposes (eg. <ul>, <footer>, <p>). There are also generic elements that don't represent any specific type of content.

Span

Tag(s):
<span></span>
Category:
Inline
Acceptable content:
Text and inline elements

Span does not inherently represent anything. It can be used to group elements for styling purposes (using the class or id attributes). It should be used only when no other element is appropriate.

Div

Tag(s):
<div></div>
Category:
Block
Acceptable content:
Everything.

Short for the word "division", divs are used to group content so it can be easily styled using the class or id attributes. It should only be used when other elements do not apply.

Generic Elements

If you can put content in an element that describes its own purpose, that should be your first choice. But if you need an element purely for the sake of styling (presentation) then divs and spans are great tools.

Wrappers

Using a div as a wrapper to which styles are added is a great example of this. The site would still function without the div element, it just wouldn't look as nice.

Wrappers are a good way to center content if the screen width is wider than your content.

div.wrapper {
  width: 100%;
  max-width: 80%;
  margin: 0 auto;
}

Un-styling Lists

Lists are great for organizing navigation links from a semantic perspective, but they don't look great straight out of the box. CSS to the rescue!

Un-styling Lists

Aside from the list item bullets, lists (both ordered and unordered) come equipped with margin and padding.

We can get rid of all three of these properties like so:

ul {
  margin: 0;
  padding: 0;
  list-style-type: none;
}

Un-styling Lists

From there, the possibilities are endless. You could use flex to line them up horizontally:

ul {
  margin: 0;
  padding: 0;
  list-style-type: none;
  display: flex;
  justify-items: space-around;
  background: mistyrose;
}

Static Positioning

  • HTML elements are positioned static by default.
  • Static elements are positioned in the normal ﬂow of the page
  • Static elements ignore top, bottom, right or left property specifications.

Static Positioning

In normal flow, inline boxes flow from left to right, wrapping to next line when needed.

<img src="img/cookie1.png"/>
<img src="img/cookie2.png"/>
<img src="img/cookie3.png"/>

Static Positioning

In normal flow, block boxes flow from top to bottom, making a new line after every box.

<p>Greetings</p>
<p>Hello</p>

Relative Positioning

  • Takes the element out of the normal flow, allowing it to be moved to the top, left, right or bottom.
  • Does not affect the elements surrounding it.
  • Makes an element a "positioning context" in which to position other elements relative to it.
  • Relative positioning and absolute positioning are used together.

Relative Positioning

The "relative" value will still put the element in the normal flow, but then offset it according to top/left/right/bottom properties. 

.relative{
  position: relative;
  left: 80px;
  top: 20px;
  height: 100px;
  background-color: yellow;
}
Hello, hi!

Absolute Positioning

  • Positions element outside of the normal flow.
  • An absolutely positioned element is offset from its container block, positioned relative.
  • Its container block is the first element that has a position other than static.
  • If no such element is found, the container block is <html>.
  • Other elements act as if it's not there.
  • Determined by its oﬀset values in the properties top, bottom, right and left.

Absolute Positioning

The "absolute" value will take the element out of the normal flow and position it in relation to the window (or the closest non-static element).

.top{
  position: absolute;
  top: -40px;
  right: 10px;
  background-color: yellow
}
.bottom{
  position: absolute;
  bottom: -40px;
  left:60px;
  background-color: green
}

Z-index

Sometimes elements overlap. You can change the order of overlapping with z-index. The element with highest z-index goes on top.

.bottom{
  position: absolute;
  bottom: 10px;
  left:60px;
  background-color: yellow;
}
.top{
  position: absolute;
  bottom: 15px;
  left:60px;
  background-color: green;
  z-index: 2;
}

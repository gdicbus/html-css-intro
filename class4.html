Measurement Units

So far in this course, we have used pixels (px) for most of our measurements. Pixels are what the browser calculates when it renders elements on the screen. But we have access to different units that provide more expressive ways to tell the browser how many pixels to draw.

Let's learn about a few of them.

Measurement Units

em

The em unit is based on the font-size set on the body. By default (in most browsers) that value is 16px, thus:
1em = 16px
1.25em = 20px
1.5em = 24px
2em = 32px
2.5em = 40px
etc.

Measurement Units

em

If, in our CSS we change the font-size value set on the body, then the math changes accordingly.

body {
  font-size: 20px;
}
1em = 20px
1.25em = 25px
1.5em = 30px
2em = 40px
2.5em = 50px

Measurement Units

Percentages

When we use a percentage as a value, the resulting calculation is based on the parent element. They allow us to leave the measurement math up to the browser.

Measurement Units

Percentages

The browser finds the size of the parent, regards that size as 100% and calculates the size of the child accordingly.

main {
  width: 1080px;
}

p {
  width: 80%;
}
<main>
  <p>Hello world!</p>
</main>

The paragraph's width would be 864px.

Width & Height

We can use our new measurement units to set specific heights and widths on block elements.

Width

An element's width describes the horizontal measurement of its content. Padding and border are additional.

header {
  width: 100%;
  padding: 1%;
}

The overall width of this header would be 102%.

Box-Sizing

content-box

This additive effect is caused by a CSS property called box-sizing, whose default value is content-box. This means that the size of the box (in this case, the width) is going to be applied to the content.

Box-Sizing

content-box

There are two ways to deal with calculating an element's width and padding. You can do the mental math and apply the width and padding accordingly.

Ex: if you want a 100px wide paragraph with padding and a border...

p {
  border: 2px solid blue;
  padding: 10px;
  width: 76px;
}

Width (content) (76px)
+ padding (20px)
+ border (4px)
= 100px

Box-Sizing

border-box

Or you can reset the box-sizing property altogether! The border-box value tells the browser to apply the width to the combination of the border, padding, and the width.

header {
  box-sizing: border-box;
  width: 100px;
  border: 2px solid blue;
  padding: 10px;
}

Width (100px)
- padding (20px)
- border (4px)
= content (76px)

Height

An element's height describes the vertical measurement of its content (unless box-sizing is set to border-box as we just discussed). By default, an element will expand to be as tall as necessary for its content (the default height value is auto).

Height

But we can also set a specific height on an element. If the contents of an element exceed its given height, they will overflow.

div {
  height: 200px;
  border: 1px solid red;
}

Overflow

When an element's content overflows its boundries, we can set the overflow property to handle it.

div {
  height: 100px;
  border: 1px solid red;
  overflow: visible;

Generic Elements

We've learned about lots of elements that were created for very specific purposes (eg. <ul>, <footer>, <p>). There are also generic elements that don't represent any specific type of content.

Span

Tag(s):
<span></span>
Category:
Inline
Acceptable content:
Text and inline elements

Span does not inherently represent anything. It can be used to group elements for styling purposes (using the class or id attributes). It should be used only when no other element is appropriate.

Div

Tag(s):
<div></div>
Category:
Block
Acceptable content:
Everything.

Short for the word "division", divs are used to group content so it can be easily styled using the class or id attributes. It should only be used when other elements do not apply.

Generic Elements

If you can put content in an element that describes its own purpose, that should be your first choice. But if you need an element purely for the sake of styling (presentation) then divs and spans are great tools.

Wrappers

Using a div as a wrapper to which styles are added is a great example of this. The site would still function without the div element, it just wouldn't look as nice.

Wrappers are a good way to center content if the screen width is wider than your content.

div.wrapper {
  width: 1400px;
  max-width: 80%;
  margin: 0 auto;
}

Display

By default, elements are either block or inline. But those defaults are not permanent! We can change them via the CSS property display.

Display

Anchors default to inline, which is how they can flow with other text.

But we can reset the display and they'll take up a whole line, just like that.

a {
  display: block;
  background: lightblue;
}

Display

Display also accepts the following values:

  • inline-block: keeps the flow of an inline element and adds the Box Model functionality you'd expect from a block (including width and height).
  • none: hides the content of an element without removing the underlying HTML.
  • grid: this inconspicuous value opens a door into the bright, shining future of web development. We don't have time to cover it, but you should definitely check it out.

Un-styling Lists

Lists are great for organizing navigation links from a semantic perspective, but they don't look great straight out of the box. CSS to the rescue!

Un-styling Lists

Aside from the list item bullets, lists (both ordered and unordered) come equipped with margin and padding.

We can get rid of all three of these properties like so:

ul {
  margin: 0;
  padding: 0;
  list-style-type: none;
}

Un-styling Lists

From there, the possibilities are endless. You could use flex to line them up horizontally:

ul {
  margin: 0;
  padding: 0;
  list-style-type: none;
  display: flex;
  justify-items: space-around;
  background: mistyrose;
}

Fun Fancy Stuff!

Backgrounds

We can do more with element backgrounds than just set them to one color!

Backgrounds

Images

Add background images to elements using the background-image property.

body {
  background-image: url(path/to/image.jpg);
}

You can use images in your file system or link to images that are hosted elsewhere by providing the full, absolute URL.

body {
  background-image: url(http://www.example.com/img/cat.png);
}

Backgrounds

Images: Repeating

By default, background images repeat like tiles. If the image is big, you may not be able to tell that it's repeating. Remove or alter this behavior using the background-repeat property.

Remove the repeat:

body {
  background-image: url(img.jpg);
  background-repeat: no-repeat;
}

Repeat horizontally:

body {
  background-image: url(img.jpg);
  background-repeat: repeat-x;
}

Repeat vertically:

body {
  background-image: url(img.jpg);
  background-repeat: repeat-y;
}

Repeat but don't crop:

body {
  background-image: url(img.jpg);
  background-repeat: round;
}

Backgrounds

Images: Size

We can alter a background image's size using the background-size property.

background-image: url(img/cat.png);
background-size: 300px;

The background image would be 300px wide with a height of "auto".

background-image: url(img/cat.png);
background-size: 100% 300px;

This background image would be 100% the width of the element and 300px high.

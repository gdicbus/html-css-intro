So, what is HTML?

Hyper Text Markup Language

Hypertext Markup Language (HTML) is the standard markup language for creating web pages
Source: Wikipedia

Invented by Sir Tim Berners-Lee in the early 90's, HTML was initially created as a way to share scientific papers. It is now governed and maintained by the good people of the W3C.

What is a web page?

A structured document that is viewed with a Web browser on a computer or mobile device screen.

How does a website get to a device?


Image credit: David Vignoni
This is an extremely simplified explanation, but it will suffice. Websites that we view on the internet are "hosted" on a server. There are tons of servers out there in the world, but their function is the same: they store data and are connected to the internet. The devices we use to view the internet, whether they're phones, laptops, or desktops, are all considered "clients" when it comes to the web. They are also connected to the internet. When you type in a website address or click a hyperlink within a web browser, the device asks for the data associated with what you're trying to find. The server "serves" it to the device, and the browser displays the result!

Your Website

A website is a way to present your content to the world. HTML allows you to structure the content in a meaningful way and CSS allows you to customize and enhance the appearance of the content.

Anatomy of a website

Concrete example:

  • The words in a paragraph are your content. 
    How to paint. That's easy. What to paint. That's much harder.
  • Putting your content into an HTML tag to make it look like a paragraph is structure. 
    <p>How to paint. That's easy. What to paint. That's much harder.</p>
  • Making the text of your paragraph green, 24px in size, and italic is presentation.

    How to paint. That's easy. What to paint. That's much harder.

HTML Elements

An element is an individual piece of information that is displayed by a browser. Websites are comprised of many elements. Browsers will present display elements in specific ways.


Element examples:

  • Paragraph
  • Heading
  • Anchor (aka Hyperlink)
  • Table
  • Image

... and many more.

HTML Tags

The code written to tell a browser which HTML Elements to display.


The following code... 

<button>I'm a button.</button>

... creates a button that can be clicked.

Basic HTML Tag Anatomy

Container Elements

These elements wrap around content, including other elements.

  • A paragraph is represented by <p> tags: 
    <p>Once upon a time there was a paragraph.</p>
  • An anchor (the element that lets us click on hyperlinks) is represented by <a> tags: 
    <a href="www.google.com">Go to Google</a>
  • Text that is emphasized (typically shown in italic) is represented by <em> tags: 
    She <em>really</em> likes cats.

Stand-Alone Elements

An element that doesn't wrap around any other content. They are "self-closing", which means there is no closing tag. Note the location of the / character in these tags:

  • An image tag: 
    <img/>
  • A horizontal rule tag: 
    <hr/>
  • A line break (like hitting the enter key in a word processer to jump down one line) tag: 
    <br/>

Element Attributes

All HTML tags accept "attributes," which provide additional information and configurations.

They are placed inside an opening tag, after the tag name.

Attributes consist of a name and a value, and the value must be contained inside quotation marks.

<input type="text" placeholder="Enter your name here." />

<a href="http://girldevelopit.com">Girl Develop It</a>

<img src="cat-photo.jpg" alt="Image of an adorable cat." />
Some elements, like img, are useless without attributes!

The HTML Must-Have Elements

Websites are made up of many different combinations of elements and content. However, there are a few tags that every website must have in order to be a real webpage.

The !Doctype Element

The first thing in an HTML document is the <!Doctype> declaration, which tells the browser which version of the markup language the page is using. Essentially, it tells the browser how to display the following code.
In our case we want it to display HTML. 

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN " "http:// www.w3.org/TR/html4/loose.dtd ">
* HTML 4.01 is outdated now. But if, for any reason, you need to use an older version of HTML, you can tell the browser so via the !Doctype. 
<!DOCTYPE html>
* This tag is case-insensitive, which is why you'll see it capitalized in various ways around the web, and probably even within these slides.

The HTML Root Element

After <!Doctype>, the entirety of a page's content must be contained within <html> tags. This is the element where we can declare which language is used for the majority of the written content of the page.

<!DOCTYPE html>
<html lang="en">

</html>
* In this case, we're using en for English. The codes de (German), fr (French), and es (Spanish) are also available, as well as many others.

Head and Body Elements

Head: The head contains the <title> of the page & meta information about the page. Meta information is not visible to the user, but has many purposes, like providing information to search engines.

Body: The body contains the actual content of the page. Everything that is contained in the body is visible to the user via their browser.

These are somewhat strange to conceptualize, so here's a metaphor that I hope will help. The HEAD of a webpage contains the mental, telepathic brainwaves that it sends to the browser and to search engines. None of the contents of the HEAD show up on the webpage itself. They're rendered elsewhere, either in the browser's chrome (like the tab where the title appears) or in a search engine. Eventually we'll link things like fonts and styles via the HEAD, as well. The BODY, meanwhile, is what the browser actually renders: the visible portion of the website.

Head and Body Elements

The HTML Must-Have Elements

Here it is, the bare minimum of a website!
Each of these elements are only declared once, eg. a webpage can only have one doctype, html root, head, title and body. 

<!DOCTYPE html>
<html lang="en">
  <head>
    <title>Title of the page</title>
  </head>
  <body>
    The page content here.
  </body>
</html>

Nesting

Container elements can wrap around other elements, the result is referred to as nesting.

For example, you could put a <p> inside of the <body> element. The <p> is now nested inside the <body>. 

<body>
  <p>A paragraph inside the body tag</p>
</body>

Nesting Order

Whichever element OPENS first CLOSES last

Photo credit: James Jordan cc

Nesting Example

Indenting your nested elements is helpful. While the browser doesn't care about indentions, the spacing is a helpful visual tool for you to understand the structure of your code! 

<html lang="en">
  <body>
    <p>I'm nested three levels</p>
    <ul>
      <li>I'm nested four levels</li>
    </ul>
  </body>
</html>

Meet the Elements

Block vs Inline

Block elements start on a new line, force elements after them to start on a new line, and take up as much width as they can.

I am a block-level paragraph.

I, too, am a block-level paragraph.

Inline elements "flow in line" with their surroundings and only take up as much space as they need.

Inside of this paragraph we have a span and an anchor, both of which are inline elements.

Inline

Text and text-related inline elements.

Browsers will typically render emphasized text in an italic font. Screen readers will change the emphasis of the vocals, like you might if you were reading aloud.

The trees were very happy.

Strong

Tag(s):
<strong></strong>
Category:
Inline
Acceptable content:
Text and text-related inline elements.

Strong is for content that is of "strong importance," including things of great seriousness or urgency (such as warnings). Browsers will typically render strong text in bold.

<strong>Warning!</strong> Railroad crossing ahead.

Warning! Railroad crossing ahead.

Anchor

Tag(s):
<a></a>
Category:
Inline
Common attributes
href, target, & title
Acceptable content:
The text that will be clickable and text-related inline elements.

Not to be confused with the <link> element! Anchors are used to create the hyperlinks you're used to seeing on the internet. Browsers tend to show them as blue, underlined text.

Here's a hyperlink to <a href="www.anotherwebsite.com">another website</a>.

Anchor Attributes

href tells the browser what path to follow when the anchor is clicked/tapped.

<a href="www.google.com">Google</a>
<a href="mailto:caitlin@csteinert.com">Email me!</a>

target tells the browser where to open the hyperlink; this one would open in a new tab/window.

<a href="www.google.com" target="_blank">Google</a>

title describes the purpose the hyperlink. On computers, it renders as a tooltip seen by hovering with a mouse. It is also read aloud by screen readers.

<a href="www.google.com" title="Navigate to Google's website">Google</a>

Image

Tag(s):
<img/>
Category:
Inline
Common attributes
alt, src, & title
Acceptable content:
None, only attributes.

The img tag is self-closing, so it cannot contain any nested elements. If the source path ( src) provided is incorrect, the browser will show a sad little broken image:

<img src="cat.png" alt="Photo of a cat" title="Photo of a cat"/>

Image Attributes

alt is text that will be displayed if, for whatever reason, the image doesn't load.

<img src="" alt="This image is super broken"/>

src tells the browser where to look for the image file.

<img src="some/folder/somewhere/cat.png"/>

title describes the purpose the image. On computers, it renders as a tooltip seen by hovering with a mouse. It is also read aloud by screen readers.

<img src="cat.png" title="A cat being absolutely adorable"/>

Path Types: Absolute

Say this website was live on the internet at www.my-website.com.

If we wanted to add a hyperlink to calico.html on index.html, we could do so using an absolute path, which includes the domain name.

Any page or image published on the internet can be linked this way.

<a href="http://www.my-website.com/cats/calico.html">See a Calico</a>

Path Types: Relative


While our websites still live on our computers, we have to use relative paths.

If we wanted to put the image calico.png on the page calico.html, it would look like this:

<img src="../../images/cat-photos/calico.png"/>

Each instance of ../ means "ascend one directory".

Path Types: Relative


A relative hyperlink on page index.html to calico.html looks like:

<a href="cats/calico.html">Calico<a>


And adding the image pitbull.png to the page index.html looks like:

<img src="images/dog-photos/pitbull.png"/>

Comments

Comments are not displayed by the browser. They are only visible when viewing the code itself.

<!-- Comment content goes here -->

Comments can be used to explain the purpose of sections of code for clarity.

<!-- Copyright Start -->
<p>Contents © of Acme Co.</p>
<!-- Copyright End -->

They can also be used make large chunks of code hidden from the browser.

<!--
<ol>
  <li>List Item</li>
  <li>Another List Item</li>
</ol>
-->

This is commonly referred to as "commenting out" code.

Escaped Characters

HTML is written in the same set of characters that use to write in English. When we need to distinguish between characters meant to be read by the browser and those meant to be read by humans, we can use escaped characters. Here's a list of all of them.

Commonly escaped HTML characters:
  • Quote: &quot; (")
  • Less-than: &lt; (<)
  • Greater-than: &gt; (>)
Other useful escaped characters:
  • Copyright symbol: &copy; (©)
  • Inverted exclamation: &iexcl; (¡)
  • Non-breaking space: &nbsp; ( )

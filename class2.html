A website is a way to present your content to the world, using HTML and CSS to present that content & make it look good.

CSS: What is it?

Cascading Style Sheets

A "style sheet language" that lets you effect how the elements on your page are displayed.

CSS is works in conjunction with HTML, but is not HTML itself.

The "cascading" aspect is very important, and we will cover that later in class.

CSS: What can it do?

Give color to text and areas, add background images, show and hide elements, and so much more!

CSS: What does it look like?

Connecting CSS to HTML

There are three ways to connect CSS to HTML. Each is useful in certain situations.

  • Inline
  • Embedded
  • External

Connecting: Inline CSS

Although CSS and HTML are seperate languages, CSS can be written in HTML files.

When we write "inline" CSS, we write it directly onto individual HTML elements within an HTML file.

Connecting: Inline CSS

The inline method allows you to add styles to a specific element using the style HTML attribute.

<body style="color: red;">
  All the text on this page is red.
</body>

<img style="border-width: 1px;" src="cat.png" />

<p style="margin-top: 99px;">
  A paragraph with lots of space above it.
</p>
  • Contents of the style attribute need to be duplicated on any other element that requires similar styling.
  • Difficult to override via the "embedded" and "external" connecting methods.

Connecting: Embedded CSS

Like inline CSS, embedded CSS is writtin within the HTML file it applies to.

Embedded CSS is stored in a <style> element within the head element.

HTML Element: Style

The contents of a style element modify the presentation of the page via CSS. There should only be one style element, and it must be within the head.

Tag(s):
<style></style>
Category:
Metadata
Acceptable content:
CSS
<!Doctype html>
<html>
  <head>
    <style>
      p {
        color: blue;
      }
    </style>
  </head>
  <body>...</body>
<html>

Connecting: Embedded CSS

<style>
  body {
    background: thistle;
  }

  p {
    margin-top: 10px;
  }

  img {
    border-width: 1px;
  }
</style>
  • Any paragraph or image on this page would have these styles applied to them automatically.
  • Embedded CSS only applies to the HTML page it is embedded within.

Connecting: External CSS

CSS can be written in a separate (external) .css file and then linked to one or many HTML files.


To create CSS files, we name them with the .css extension when we save.

HTML Element: Link

Tag(s):
<link>
Category:
Metadata
Common attributes:
href, rel

The link element (not to be confused with the anchor!) allows us to define a path to our stylesheet within the head of the HTML page.

Note that although this is a self-closing element, it does not need the / at the end.

<link rel="stylesheet" href="style.css">

Connecting: External CSS

The external CSS file is linked within the HTML's head using a <link> element:

<head>
  <link rel="stylesheet" href="awesome-styles.css">
</head>

Linking to external CSS has been the industry-standard approach for many reasons:

  • They create style resources that can be shared across multiple pages.
  • They reduce file size & bandwidth.
  • They are easier to maintain in larger projects than inline and embedded styles.

Let's Develop It!


Create a new file in your editor and save it as styles.css. Make sure this new CSS file is in the same folder as your HTML file from last class!

Copy/paste the following code into your new CSS file and save:

body {
  background-color: thistle;
}

Next, add a <link> in the head of your HTML page and connect to your new CSS file. Save and refresh the page!

Anatomy of the CSS Rule

Anatomy of the CSS Rule

The typical CSS file contains many rules.

h1 {
  display: block;
  text-align: center;
}

main {
  font-weight: bold;
}

footer {
  background-color: red;
  color: black;
}
  • The rule starts with a selector.
  • Once the selector is stated, we open the rule with a curly brace ( { )
  • Rules contain sets of properties and values .
  • The rule ends with a closing curly brace ( } ).

Anatomy of the CSS Declaration

A property - value pair is called a declaration.

nav {
  display: flex;
  flex-direction: row;
  flex-wrap: nowrap;
}
  • The property must be followed by a colon ( : )
  • The value must be followed by a semicolon ( ; )
  • In order to be read easily (by humans) it's best for each declaration to be on its own line.

Pop Quiz!

In the CSS below:

  2. Which is the selector?
  3. Which is the property?
body {
  background-color: thistle;
}

  2. body
  3. background-color

CSS Color Values

CSS accepts many different color value types. All of the following are the same color.

  • Color name: thistle
  • Hexadecimal value: #D8BFD8
  • RGB value: rgb(216, 191, 216)
  • HSL value: hsl(300, 24%, 80%)


Here's how they look as a background-color declaration:

background-color: thistle;
background-color: #D8BFD8;
background-color: rgb(216, 191, 216);
background-color: hsl(300, 24%, 80%);

CSS Selectors

Selectors allow us to match our styles to the HTML elements we want to apply them to.

Let's meet the different types of selectors we can use.

Element Type Selectors

We can use the tag name of an element within CSS to apply styles. Say we have the following in our HTML:

<span>Light blue is my favorite color.</span>

We can apply a background color via our CSS assigning a background-color style to span element types.

span {
  background-color: lightblue;
}

It would look something like this: Light blue is my favorite color.

Element Type Selectors

Note that when you set a style for a particular element type, it will apply to every instance of that element.

span {
  background-color: lightblue;
}
<span>Light blue is my favorite color.</span>
<span>Red is my favorite color.</span>
<span>Light green is my favorite color.</span>
<span>Purple is my favorite color.</span>

Light blue is my favorite color. Red is my favorite color. Light green is my favorite color. Purple is my favorite color.

Let's Develop It!

Let's toss in a new CSS property: color. The color property changes the color of the text within an element.

span {
  color: darkred;
}

Use the Element Type Selector method we just learned about to change the color of the text inside paragraphs.

Descendent Selectors

We can select HTML elements for styling based on how they're nested within other elements.

Descendent Selectors

Multiple selectors separated by spaces tell the browser to get specific before applying a style.

In this case, the browser knows that if it finds an unordered list containing a list item containing an anchor, it will apply this background-color to the anchor.

ul li a {
  background-color: lightblue;
}

The style is always applied to the right-most item in the selector list.

Class & ID Selectors

Element types aren't the only identifiers for assigning styles. We can also use class and id attributes as identifiers.

Class Selectors

All HTML elements accept class attributes. As the developer, you decide what any given class does, how it's named, and which elements to apply it to.

CSS:

Create a rule for your class. Classes are signified by a . (period), and then the class name.

.fancy {
  color: white;
  background: coral;
  font-weight: bold;
  font-style: italic;
}

HTML:

Add the class attribute to your element(s) and assign the value (the class name).

<p class="fancy">
  A fancy paragraph!
</p>
<span class="fancy">
  A fancy span!
</span>

Class Selectors

Multiple classes can be added to a single element. Just make sure there are spaces between them!

CSS:

.bold {
  font-weight: bold;
}

.italic {
  font-style: italic;
}

HTML:

<p class="bold italic">
  Bold & italic text.
</p>

<p class="bolditalic">
  There is no such class.
</p>

ID Selectors

All HTML elements accept one id attribute. As the developer, you decide what any given id does, how it's named, and which element to apply it to.

CSS:

Create a rule for your id, signified by a # (pound, hashtag) followed by the id name.

#logo {
  color: darkred;
  letter-spacing: 2px;
  font-weight: bold;
}

HTML:

Add the id attribute to your element and assign the value (the id name). 

<h1 id="logo">
  Netflix
</h1>

ID Selectors

Unlike class selectors, id selectors must be unique: an id should only appear once per HTML page. Elements can only be assigned one id at a time.


Ids can also be used for navigation linking within an HTML page:

<a href="#party">Clicking this link...</a>
<br/>
<br/>
<br/>
<br/>
<br/>
<br/>
<br/>
<h3 id="#party">...would jump to here.</h3>

Cascading Style Sheets

Photo credit: Tristan Schmurr

The Cascade

The styles applied to an element can come from many different sources. It is common to have multiple styles conflict.

How does the browser know which one to display?

CSS resolves these conflicts by assigning a weight to each style rule and when several rules apply, choosing the one with the greatest weight.

- W3C cascade documentation

The Cascade

Within a CSS Rule: Last Wins

If an element is assigned the same style value twice (or more) within a rule, the last value will be the one applied by the browser.

p {
  color: blue;
  font-weight: bold;
  font-style: italic;
  letter-spacing: 2px;
  color: magenta;
}

In this example, the browser would display paragraph text's color as magenta.

Pro Tip: organize the rules within CSS properties consistantly so as to reduce the chances of accidentally repeating them.

I recommend alphabetical order, personally.

The Cascade

Within a Style Sheet: Last Wins

If an element is assigned the same style value twice (or more) within one style sheet, the last value will be the one applied by the browser.

p {
  color: blue;
}

span {
  letter-spacing: 2px;
}

p {
  color: magenta;
}


In this example, the paragraph text color would be magenta in the browser.

The Cascade

External CSS Import Order: Last Wins

If more than one CSS file is linked to an HTML page, style conflicts will be resolved based on which file was imported last. 

/* other-styles.css */
p {
  letter-spacing: 2px;
}
/* some-styles.css */
p {
  letter-spacing: 5px;
}

<!-- page.html -->
<head>
  <link rel="stylesheet" href="some-styles.css">
  <link rel="stylesheet" href="other-styles.css">
</head>

In this case letter-spacing:2px; would "win".

The Cascade

CSS Closest to the HTML Wins

Remember the three ways to connect styles? There's an important "gotcha" to know about:

Styles within an HTML file will win over external styles.

<head>
  <link rel="stylesheet" href="some-styles.css">
  <link rel="stylesheet" href="other-styles.css">

  <style>
    p {
      letter-spacing: 100px;
    }
  </style>
</head>

In this case it doesn't matter what order the external files are linked, letter-spacing:100px; would "win".

The Cascade

Inline Styles Win

And finally, styles that are written inline on an individual HTML element win over all of these other styling methods.

<head>
  <link rel="stylesheet" href="styles.css">
  <style>
    p {
      letter-spacing: 100px;
    }
  </style>
</head>
<body>
  <p style="letter-spacing: 1px;">
    Inline styles win in this instance.
  </p>
</body>

Let's Develop It!

  • Duplicate properties within a CSS rule. Change their values and see which gets applied.
  • Duplicate (and slightly change) entire rules within your CSS file and see which gets applied.
  • Add an inline style to an HTML element and use it to override existing styles.
  • Create a new CSS file and link it in the head of your HTML. Swap the order of the links.
  • Try the letter-spacing CSS property! 
    span {
  letter-spacing: 2px;
}

The Cascade: Specificity

The way in which CSS is connected to HTML isn't the only method CSS uses to determine what styles should be applied. CSS also assigns "weight" to different types of selectors. The selector (or group of selectors) that is the "heaviest" wins.


From lightest to heaviest:
element type selector
class
id
inline styles
!important

Selector Specificity

Element Type Selectors

Element type selectors are the lightest of the bunch, but they can override each other when they are combined.


body p {
  font-size: 28px;
}

p {
  font-size: 18px;
}

Because the first rule uses a descendant selector, it is more specific than the second one, so it wins.

Selector Specificity

Class Selectors

Adding a class to an element and assigning styles to the class will override any matching styles that are being applied via element type selectors.

<span class="my-span">Happy trees.</span>
.my-span {
  font-size: 18px;
}

span {
  font-size: 28px;
}

Even though the class comes first in this CSS example, it would win because CSS assigns more weight to a class than an element type selector.

Selector Specificity

ID Selectors

CSS gives more weight to ids than classes. When an element has both, the id styles will win.

#span-id {
  font-size: 18px;
}

.span-class {
  font-size: 28px;
}
<span class="span-class" id="span-id">Hello!</span>

The result here would be a paragraph with a font-size of 18 pixels.

Selector Specificity

Inline Styles

CSS gives more weight to inline CSS than it does to ids, classes, or element type selectors. This "heaviness" is part of the reason why it's a best-practice not to use inline styles. 

#span-id {
  color: black;
}
<span style="color: magenta;" id="span-id">Hi!</span>

The result: Hi!

Selector Specificity

The Nuclear Option: !important

The only way to override an inline style (without changing the inline style itself) is to use the !important flag.

If we take the previous example:

<span style="color: magenta;" id="span-id">Hi!</span>

And append !important to the color value in our CSS file, then it will win. Even if we use a "light" element type selector.

span {
  color: black !important;
}

Selector Specificity

The Nuclear Option: !important

The !important flag is very difficult to override once it's been used. For that reason, it should be used only as a last ditch effort, when you have absolutely no other options.

Specificity

Specificity is one of the trickiest concepts to master in CSS, so do not fret if you're confused. Time and practice (and asking for help!) will aid in conquering this concept!

CSS Property: Font-family

This property allows us to set the font of our text.

We can set it to a generic category and let the browser pick a font that qualifies: 

body {
  font-family: serif;
}
Serif fonts have the little nubbins (serifs) at the tips of each letter. 
h1 {
  font-family: sans-serif;
}
Sans-serif (literally "without serif") fonts have no little nubbins at the tips.


Other generic font types include:
cursive, fantasy, and monospace.

CSS Property: Font-family

We can also specify a named font. If the name has more than one word, it's safest to wrap it in quotes.

font-family: Georgia;
Georgia 
font-family: Arial;
Arial 
font-family: Courier;
Courier 
font-family: Papyrus;
Papyrus 
font-family: Impact;

(may not be available on PC) 
font-family: "Comic Sans";
Comic Sans
(may not be available on Mac)

Note: The browser can only display fonts that are on the device. Here's a helpful list of fonts by platform.

CSS Property: Font-family

The safest route is to provide a list of fonts, separated by commas:

body {
  font-family: "Big Caslon", Georgia, serif;
}

The browser will read through the list from left to right, apply the first font that it has access to, and will then ignore the rest.

Property: Font-size

The font-size property specifies the size of the font. In this example we are using the px unit (pixels) to calculate the font size. There are lots of other ways to describe font-size. We'll learn about other units later.

.very-large-text {
  font-size: 100px;
}

Very Large Text

.very-small-text {
  font-size: 10px;
}

Very Small Text

Property: Font

There are a lot of CSS properties relating to font. Luckily, CSS allows us to write them in "shorthand," that is put them all together with the font property.

p {
  font-style: italic;
  font-weight: bold;
  font-size: 10px;
  font-family: Calibri, sans-serif;
}

... can also be written as...

p {
  font: italic bold 10px Calibri, sans-serif;
}

Property: Font

Note: there are some "gotchas" with this property. The font-family and font-size values are mandatory. Also, the font-family value must be last.

The following code would be ignored by the browser because:

p {
  font: Helvetica italic bold 18px;
}

The font-family isn't listed last.

p {
  font: italic bold Helvetica;
}

There's no font-size value.

Let's Develop It!

  • Set the font-family on the body element to see how it effects the entire page.
  • Apply a different font-family and/or font-sizes on specific elements on your page.
  • Try out font-style (more info here) and font-weight (more info here) properties.
